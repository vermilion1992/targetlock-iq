# Scenario lab guide (pilot testers)

**In app:** Top bar → **Scenario lab**  
**In guide:** Standard workflow step 17 · Branch workflow step 2 · Quick orientation step 6

During a **Guide**, use **Load demo for this step** when offered — data restores on **Exit guide**. Opening Scenario lab directly still replaces the active hole until you switch holes or restore from a hole package.

| Tab | Use |
|-----|-----|
| Built-in | Seven synthetic single-hole cases (on-plan, drift, correction, motor review, wedge, QA jump, invalid CSV) |
| Custom | Build a hole from dip/azimuth/drift parameters |
| Branch programs | Ten demos (five Phase 1 + five Phase 2) — loads **persisted** mother + daughter library |

Full matrix: [test-scenarios.md](../test-scenarios.md) · [guide-center.md](./guide-center.md)

**Note:** Scenario loads show a **Simulated** badge; exports record the scenario name when active.
